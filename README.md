# 秘马群 Dota2 内战小程序

微信原生云开发小程序。每位用户通过微信身份维护自己的选手卡，报名、候补、分队、积分榜和战绩都保存在同一个云环境中。

积分与自评分是两套独立数据：选手自己填写自评分，系统自动分队时使用自评分；比赛积分从 0 开始，只按胜负计算，胜者 `+2`，负者 `-1`。

## 项目结构

这是根目录小程序项目，不使用嵌套的 `miniprogram/` 目录：

- `app.json`、`pages/`、`utils/`：小程序端。
- `cloudfunctions/api/`：业务 API 和数据库写入。
- `cloudfunctions/leagueSync/`：OpenDota 联赛定时同步。
- `tests/`：Node.js 自动测试。

在微信开发者工具中导入本目录，并使用项目对应的小程序 AppID。

## 云开发配置

1. 在微信开发者工具中开通或选择一个云环境。
2. 在 `utils/config.js` 中填写该云环境 ID。
3. 确认 `project.config.json` 的 `cloudfunctionRoot` 为 `cloudfunctions/`。
4. 云端环境中需要使用 `players`、`rooms`、`matches`、`system` 和 `leagueSyncQueue` 集合；API 首次运行时会尝试初始化所需集合和文档。

## 联赛自动同步

自动同步每 15 分钟运行一次，当前包含：

- 秘马内战，League ID `20040`。
- 斐济杯，League ID `19608`。

两个联赛都会读取 OpenDota 联赛列表。斐济杯还会使用 Valve 联赛历史接口发现比赛，并用比赛 `8900989622` 作为首次同步的幂等校验种子。完整且能唯一关联到 10 张选手卡的 5 对 5 比赛会自动入库；缺少选手、Steam ID 重复归属或阵容不完整时只进入管理员确认队列，不会自动修改积分。

同一个比赛 ID 无论被 OpenDota、Valve 或校验种子发现多少次，都只会结算一次。

### 云函数环境变量

在云开发控制台分别打开 `api` 和 `leagueSync` 云函数的配置页：

1. 在两个云函数中都新建 `LEAGUE_SYNC_TOKEN`。
2. 两处填写完全相同的随机值，长度至少 32 个字符。
3. `STEAM_WEB_API_KEY` 只配置在 `api` 云函数中，不要配置到小程序端或提交到 Git。
4. 不要把任何真实密钥写入源码、README 或截图。

### 英雄头像域名

英雄中文名保存在小程序代码内，英雄头像使用 Steam 静态图片域名，不占用小程序包大小。需要在微信公众平台的小程序后台，把下面域名加入 **downloadFile 合法域名**：

```text
https://cdn.cloudflare.steamstatic.com
```

开发者工具关闭域名校验只能用于本地调试，真机和正式版本仍需要配置合法域名。

### 上传顺序

1. 右键 `cloudfunctions/api`，选择 **上传并部署：云端安装依赖**。
2. 右键 `cloudfunctions/leagueSync`，选择 **上传并部署：云端安装依赖**。
3. 再次右键 `cloudfunctions/leagueSync`，选择 **上传触发器**。
4. 打开云开发控制台，确认 `leagueSyncEvery15Minutes` 定时触发器存在。
5. 用管理员账号进入“战绩”页，点击 **立即同步**。
6. 确认页面的“上次成功”时间发生更新，并能看到“秘马内战”或“斐济杯”的来源标签。

### 上线核对

按下面顺序验证一次真实流程：

1. 完整匹配 10 位选手的比赛只导入一次，胜方 `+2`、负方 `-1`。
2. 再次立即同步不会重复增加积分或场次。
3. 存在未知 Steam ID 的比赛进入管理员待确认列表，确认前积分不变。
4. 管理员重新关联实际阵容后可以导入一次。
5. “我的”页显示积分排名、胜负、KDA、GPM、XPM、近 20 场英雄、Steam ID 和最近比赛。
6. 从“我的”和“历史战绩”都可以打开比赛详情。
7. 打开一条旧的 API 导入战绩，确认英雄、KDA、GPM、XPM 自动补全，但排行榜积分、场次和胜场没有再次增加。
8. 手动录入的战绩显示双方自评总分，实际参赛阵容显示选手头像和当时的自评分。
9. 删除导入战绩后，相关选手的积分、场次和胜场会回滚。
10. 用第二台手机进入同一小程序，确认战绩和个人统计一致。

定时触发器和真机同步必须在微信开发者工具及云环境中验证，本地 Node.js 测试不能替代这一步。

### 队列维护

当前同步任务单次最多扫描 `leagueSyncQueue` 的 1000 条记录。联赛累计接近 1000 场前，应在确认对应 `matches` 战绩完整后，归档或清理已经不再由 OpenDota 联赛列表返回的 `imported` 终态队列记录。不要删除 `needs_review`、`waiting_data`、`failed` 或 `processing` 记录。

## 选手流程

1. 进入“我的”，填写选手名、自评分、一个或多个 Steam ID、偏好位置并保存。
2. 进入“开 C”报名；第 11 人起自动进入候补名单。
3. 满 10 人后系统按自评分和 1 至 5 号位偏好自动分队。
4. 管理员可核对实际参赛阵容、记录胜负、取消单人报名、清空本轮报名和标记鸽子。
5. 后续局优先安排上一局未参赛的候补，其余名额随机补齐。
