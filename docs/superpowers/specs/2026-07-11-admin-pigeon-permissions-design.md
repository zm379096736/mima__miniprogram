# 管理员面板与鸽子记录权限设计

## 目标

- 已配置的管理员进入“开 C”页面后稳定看到管理员面板。
- 只有管理员能够清空报名和记录鸽子。
- 普通用户看不到管理员操作入口，也无法绕过页面直接调用云函数。

## 管理员识别

前端配置与云函数使用同一组管理员 OpenID。云函数 `bootstrap` 返回当前登录用户的 `openid` 和 `isAdmin`；房间页优先使用云端 `isAdmin`，并用当前 `openid` 对本地配置做兼容校验。

房间页显示管理员身份状态。管理员显示管理面板；普通用户不显示管理操作。这样即使部署版本或缓存不一致，也能从页面状态判断身份链路是否正常。

## 权限边界

- “清空报名并重开”仅在 `isAdmin` 为真时显示，云函数再次验证调用者 OpenID。
- “鸽子统计”仅在 `isAdmin` 为真且存在报名或候补选手时显示。
- `markPigeons` 云函数必须接收当前调用者 OpenID，并在修改选手数据前验证管理员身份。
- 非管理员调用管理员操作时返回明确的“只有管理员可以操作”提示。

## 编码修复

完整重写已乱码的房间页 JS、WXML 和相关测试文件，统一保存为无 BOM 的 UTF-8，避免字符串损坏、标签闭合错误和页面渲染异常。

## 测试与验收

- 测试管理员名单中的两个 OpenID 都能识别为管理员。
- 测试普通 OpenID 不能执行鸽子记录。
- 检查 WXML 中鸽子统计区域受 `isAdmin` 控制。
- 运行全部单元测试、JavaScript 语法检查、乱码扫描和 BOM 扫描。
- 微信开发者工具中重新上传并部署 `cloudfunctions/api` 后，管理员进入“开 C”页应看到管理员面板与鸽子统计；普通用户两者都不可见。
